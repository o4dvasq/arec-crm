## Notion Meeting Sync (runs during every /productivity:update, after inbox processing)

Pull meeting notes from Notion, reformat with AREC context, extract action items, enrich KB with new contacts, and save as standalone markdown files. Deduplication ensures already-processed meetings are skipped on re-runs.

**Key design principle:** Notion AI already generates structured summaries (Action Items, Meeting Context, Key Topics, etc.) from transcripts. Do NOT re-summarize from raw transcripts. Instead, fetch the Notion-generated summary, reformat it with AREC context, and enrich the KB.

### Step 1: Query recent meetings

Use `notion-query-meeting-notes` to fetch meetings from the **last 5 days**:

```json
{
  "filter": {
    "operator": "and",
    "filters": [{
      "property": "created_time",
      "filter": {
        "operator": "date_is_within",
        "value": {
          "type": "relative",
          "value": "custom",
          "direction": "past",
          "unit": "day",
          "count": 5
        }
      }
    }]
  }
}
```

### Step 2: Deduplicate

Check `meeting-summaries/` for existing files. Summary filenames follow `YYYY-MM-DD-meeting-title-slug.md`. If a file exists for a meeting (match by date + fuzzy title on filename), skip it.

Create `meeting-summaries/` if it doesn't exist.

### Step 3: Fetch and reformat each new meeting

For each unprocessed meeting:

1. **Fetch** using `notion-fetch` with the meeting URL and `include_transcript: false`
   - This returns Notion's AI-generated summary (Action Items, Meeting Context, Key Topics, etc.) WITHOUT the full transcript
   - The summary sections appear as structured markdown in the page content
   - Do NOT use `include_transcript: true` — the transcript is redundant when the summary is available

2. **Reformat** — take Notion's structured summary and reformat into our standard briefing format. Use CLAUDE.md context to:
   - Resolve first names ("Tony" → Tony Avila, "Truman" → Truman Flynn)
   - Expand acronyms ("A&D" → Acquisition & Development)
   - Add AREC context to deal references
   - Identify the attendees from the Meeting Context or transcript metadata

   File format:
   ```
   # [Meeting Title]

   **Date:** YYYY-MM-DD
   **Source:** [Notion](url)
   **Attendees:** comma-separated, using known names from CLAUDE.md

   ## Summary
   2-5 paragraph narrative synthesized from Notion's summary sections.
   Decisions, context, next steps — not blow-by-blow.

   ## Key Decisions
   - Concrete decisions made

   ## Action Items
   - [ ] **[Person]** — Description

   ## Open Questions
   - Unresolved items
   ```

3. **Save** to `meeting-summaries/YYYY-MM-DD-meeting-title-slug.md`

### Step 3b: Contact & KB Enrichment (runs for each new meeting)

After saving the summary, scan the meeting content for **new contacts** — people mentioned who are NOT already in CLAUDE.md People table or `memory/people/`. This is critical for keeping the KB current.

**For each person mentioned in the meeting who is not a known AREC team member:**

1. **Check** if they exist in the CLAUDE.md People table (match by first name, last name, or org)
2. **Check** if an intel file exists in `memory/people/` (match by name slug)
3. **If unknown** — surface to Oscar for validation:

   > **New contact from [Meeting Title]:**
   > **[Name]** — [what we know: org, role, connection context from the meeting]
   > Want me to add them to the KB? If so, any additional context (email, role, org)?

4. **If Oscar confirms** — create or update:
   - Add a row to the CLAUDE.md People table (short name, full name / role)
   - Create `memory/people/<name-slug>.md` with:
     ```
     # [Full Name]

     ## Overview
     - **Organization:** [from meeting context]
     - **Role:** [from meeting context]
     - **Email:** [if provided]
     - **Phone:** [if provided]
     - **Type:** investor | consultant | advisor | contact
     - **Met via:** [how they connected — e.g. "Tony met at VIP conference"]

     ## Intel
     - [Key facts from the meeting — e.g. "Chair of pension fund", "board member at Alameda County ERA"]
     - [Relationship angle — e.g. "Current IB firm not a fit, but Alameda board connection is the angle"]

     ## Interaction History
     - YYYY-MM-DD — [Meeting title] — [1-line summary of interaction]
     ```
   - If they map to a CRM prospect org, update `crm/prospects.md` contact/notes fields
   - If they're a net-new investor prospect, ask Oscar if they should be added to `crm/prospects.md`

5. **If already known but meeting adds new intel** — update `memory/people/<slug>.md` with new facts and add an interaction entry

**Present all new contacts from all meetings together** (not one-by-one per meeting) so Oscar can batch respond.

### Step 4: Catch untranscribed meetings

After processing Notion transcripts, check Oscar's Outlook calendar for recent meetings that do NOT have a matching Notion transcript or existing summary file.

Use `outlook_calendar_search` to pull meetings from the **last 2 days** (or since the last update, whichever is shorter). For each past calendar event:

1. **Skip** if a meeting summary already exists in `meeting-summaries/` (match by date + fuzzy title)
2. **Skip** if a Notion transcript was just processed for this meeting in Step 3
3. **Skip** if it's a travel/logistics event (trains, flights, hotel), a personal calendar block, or an all-day event
4. **Skip** if it only has 1 attendee (Oscar alone — likely a placeholder/block)

For each remaining unmatched meeting, prompt Oscar:

> **No transcript found for: [Meeting Title]** ([date], [time], with [attendee names])
> Anything notable you want to add to the knowledge base? Any action items or follow-ups I should remember?

If Oscar provides notes:
- Save a lightweight summary to `meeting-summaries/YYYY-MM-DD-meeting-title-slug.md` using the same format as Step 3, but mark the source as `**Source:** Manual capture (no transcript)` instead of a Notion link
- Extract any action items for Step 5
- Run Step 3b contact enrichment on the notes

If Oscar says nothing notable, skip it — don't create a file.

Present all unmatched meetings together (not one-by-one) so Oscar can batch respond or dismiss the whole group.

### Step 5: Extract and propose action items

Collect all action items from new summaries (both Notion-sourced from Step 3 and manually-captured from Step 4). Cross-reference against TASKS.md (fuzzy match). Present new Oscar-owned items grouped by meeting. Wait for user confirmation before adding to TASKS.md.

### Step 6: Report

Include in the update output:
```
Meetings: N found in Notion, N new summaries saved, N already processed
Untranscribed: N calendar meetings without transcripts prompted
New contacts: N identified, N confirmed (pending Oscar review)
Action items: N extracted, N new (N already tracked)
```

While processing, read today's calendar events. For each event, check whether
the attendees or event title match a High urgency org name in prospects.md.

If a match is found, load silently:
- The prospect record from crm/prospects.md
- The last 3 interactions from crm/interactions.md for this org
- The intel file from memory/people/<org-slug>.md (cap at 800 chars if long)

Then surface a pre-meeting section **before** the CRM Pulse:

> **Pre-Meeting: [Org Name] — [Time] [Meeting Title]**
> [2-4 sentences synthesizing: where things stand, the key open question,
> what the goal of today's meeting should be, and any critical context
> from the intel file — e.g. known objections, decision timeline, who
> the real decision-maker is]
> Last touch: [N days ago] — [brief description of last interaction]

**Quality bar:** This paragraph should be specific enough to change how
Oscar prepares for the meeting. Generic summaries ("they're interested in
Fund II") are not acceptable. If there's no intel file and no meaningful
interaction history, say so rather than padding.

If no High urgency meetings today: skip this section entirely.
